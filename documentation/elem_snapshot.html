elem_snapshot() - Element Snapshots

SYNOPSIS

elem_snapshot_t: C -> *Type;

elem_snapshot: C × ℕ*E -> S;
elem_snapshot( container, elem_ref);

DESCRIPTION

If, for a portion of code, a data structure C behaves like a fixed set—that is the ordering of elements may change but none are added or removed, then elem_snapshot() can be used to temporarily save a specific node in a location independent way.

RETURN VALUE

For arrays, elem_snapshot() will return the object of type S = E indicated by elem_ref via copying. For one-elment-node based structures, elem_ref is trivially returned.

The type function elem_snapshot_t() is a compile-time concept that returns a type-pointer pointing for identifying type S.

ALGORITHM

For C ε Containers ∧
    T = element_t(C) ∧
    S = elem_snapshot_t(C), let
elem_snapshot: C × ℕ*T -> S;

For c::C, r:= ℕ*T, and return: S, if
(T = S) -> elem_snapshot(c, r) = { return <- [r]; } ∧
(T != S) -> elem_snapshot(c, r) = { return <- r; };
